# Mass Mailing System

## Описание проекта

Mass Mailing System — это веб-приложение на Django для управления рассылками и клиентами. Система предоставляет следующие возможности:

### Для пользователей:
- Создание, просмотр, редактирование и удаление своих клиентов и рассылок.
- Просмотр статистики по своим рассылкам (успешные/неуспешные попытки).

### Для менеджеров:
- Просмотр всех клиентов и рассылок.
- Просмотр списка пользователей сервиса.
- Блокировка и разблокировка пользователей сервиса.
- Отключение рассылок.

### Основные функции:
- Управление пользователями (система ролей: пользователь и менеджер).
- Автоматическая отправка сообщений с использованием Django Email Backend.
- Отчеты и статистика по каждой рассылке.
- Кэширование для улучшения производительности.

## Установка проекта

### Требования

- Python >= 3.8
- Django >= 4.0
- PostgreSQL (рекомендуется) или SQLite
- Redis (для кэширования, если используется)

### Установка

1. Клонируйте репозиторий:
   ```bash
   git clone <URL вашего репозитория>
   cd mass-mailing
   ```

2. Создайте виртуальное окружение и активируйте его:
   ```bash
   python -m venv venv
   source venv/bin/activate  # Для Linux/Mac
   venv\Scripts\activate   # Для Windows
   ```

3. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```

4. Настройте файл `settings.py`:
   - Убедитесь, что подключена база данных (PostgreSQL или SQLite).
   - Укажите параметры электронной почты:
     ```python
     EMAIL_BACKEND = 'django.core.mail.backends.smtp.EmailBackend'
     EMAIL_HOST = 'smtp.example.com'
     EMAIL_PORT = 587
     EMAIL_USE_TLS = True
     EMAIL_HOST_USER = '<ваш_email>'
     EMAIL_HOST_PASSWORD = '<ваш_пароль>'
     ```

5. Примените миграции:
   ```bash
   python manage.py migrate
   ```

6. Создайте суперпользователя:
   ```bash
   python manage.py createsuperuser
   ```

7. Запустите сервер разработки:
   ```bash
   python manage.py runserver
   ```

8. Откройте браузер и перейдите по адресу: [http://127.0.0.1:8000/](http://127.0.0.1:8000/).

## Основные URL

| URL                          | Описание                                      |
|------------------------------|-----------------------------------------------|
| `/home/`                     | Главная страница со статистикой              |
| `/mailing_list/`             | Список рассылок                              |
| `/mailing_form/`             | Создание новой рассылки                      |
| `/message_list/`             | Список сообщений                            |
| `/recipient_list/`           | Список клиентов                             |
| `/users/`                    | Управление пользователями (для менеджеров)   |

## Использование кэширования

Кэширование настроено для следующих представлений:
- Список рассылок (`/mailing_list/`): данные кэшируются на 15 минут.
- Список пользователей (`/users/`): данные кэшируются на 15 минут.

Если необходимо обновить данные кэшированных страниц, используется ручное или автоматическое сброс кэша через сигналы.
